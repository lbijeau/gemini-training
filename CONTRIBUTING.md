# Contributing to Gemini Training

We welcome contributions to the Gemini Training repository! Whether you're fixing a typo, adding a new example, or creating a whole new module, your help is appreciated.

## How to Contribute

1.  **Fork the Repository:**
    Start by forking the main repository to your own GitHub account.

2.  **Clone Your Fork:**
    Clone your forked repository to your local machine.
    ```bash
    git clone https://github.com/your-username/gemini-training.git
    ```

3.  **Create a New Branch:**
    Create a new branch for your changes. Use a descriptive name, such as `feat/add-new-exercise` or `fix/typo-in-readme`.
    ```bash
    git checkout -b your-branch-name
    ```

4.  **Make Your Changes:**
    Make your changes to the code and documentation. Please ensure your contributions align with the existing style and structure of the repository.

5.  **Commit Your Changes:**
    Commit your changes with a clear and descriptive commit message. We follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification.
    ```bash
    git add .
    git commit -m "feat: Add a new module on test generation"
    ```

6.  **Push to Your Fork:**
    Push your changes to your forked repository on GitHub.
    ```bash
    git push origin your-branch-name
    ```

7.  **Create a Pull Request:**
    Open a pull request from your fork to the main repository. In the description, please explain the changes you've made and why.

## Contribution Ideas

*   **Add New Exercises:** Create new exercises for the existing modules.
*   **Create New Modules:** Propose and create new modules on advanced Gemini topics.
*   **Improve Existing Content:** Fix typos, clarify explanations, or add more examples to the existing modules and exercises.
*   **Add More Examples:** Add more standalone examples to the `examples/` directory.
*   **Suggest a New Feature:** Open an issue to suggest a new feature or improvement.

Thank you for contributing!
